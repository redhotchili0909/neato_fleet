Neato Fleet Traffic Control

Coordinating multiple Neato robots with a centralized traffic controller that plans smooth, collision-free paths across shared spaces.

Project Vision Motivation Milestones

Project Vision

We want to build a traffic control system for robots. The core idea is to have a centralized algorithm that sees the whole map and all the robots. It will calculate the perfect paths for multiple Neatos to get to their destinations without crashing into each other, and then send those instructions to the robots to execute.
Eventually, we want to try decentralizing the system to see if the robots can figure out traffic on their own without the centralized system (racing), but we are starting with a centralized system.

Motivation

We are interested in the problem of coordination. It’s easy to make one robot move, but making three robots move in a crowded space is a huge logistical challenge. We want to see if we can code a system that is smarter than just "stop if you see something." We want the robots to weave around each other efficiently, better than an intersection with traffic lights.
Possible Applications:

  • Warehouses: Like the Amazon Kiva robots that zip around moving packages without colliding.
  • Traffic Control: Self-driving cars navigating a 4-way stop without traffic lights.
  • Swarm Shows: Drones or robots performing synchronized movements.

    • Milestone Timeline

    Milestone 1 — Control + Research

    • By milestone 1, we definitely need to be able to control all the neatos from one computer. This can be as simple as being able to send motor commands to each neato, and verifying that the neatos are moving the correct wayiver algorithm decision with math references
    • We also need to do some research into how multi-agent systems like this implement common path planning algorithms. The deliverable for milestone 1 could be a decision on what algorithm we are using with links to the math

    Milestone 2 — Centralized Execution

      By milestone 2, we need to have the centralized implementation of path planning working. This can be as simple as the neatos all having an end goal where the end goals all cross with each other, but it’s pretty much in one direction (like a drag race).

    Final — To Be Decided

      Tentative, but by the final we could either have decentralized path planning, or move advanced centralized path planning, where the neatos have a map of the room with their end goals and their end goals are not in a simple straight line.
